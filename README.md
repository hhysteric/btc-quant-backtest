# BTC 行情量化回测

纯前端的比特币策略回测工具。在同一段历史行情上，自动寻找**收益最大的 MA / EMA 择时策略**，并与**周定投 / 月定投 / ahr999 指数定投 / 买入持有**横向对比。

所有数据获取与计算都在你的浏览器本地完成，无后端、零运行成本，可直接用 GitHub Pages 部署。

## 功能

- **两种数据源**
  - Binance 公开 K 线 API（免密钥，支持日/周/月线，自动分页拉取）
  - 本地上传 Excel / CSV（兼容中英文表头：date/open/high/low/close 或 日期/开盘/最高/最低/收盘）
- **MA / EMA 参数寻优**：在设定的周期范围内遍历，列出收益排行 Top 10，自动选出最优参数
  - 单均线模式：收盘价在均线上方持币，下方空仓
  - 双均线模式：短均线在长均线上方持币（金叉/死叉）
- **定投对比**
  - 周定投 / 月定投：固定金额定期买入，只买不卖
  - ahr999 定投：每周计算 ahr999 指数，低于阈值时买入，指数越低买得越多（加权封顶 3 倍）
  - 买入持有：作为基准
- **结果可视化**：多策略资金曲线对比图（ECharts）、收益指标表、参数排行表

## 使用

直接打开 `index.html` 即可，或起一个静态服务：

```bash
python -m http.server 8000
# 浏览器访问 http://localhost:8000
```

操作流程：
1. 选数据源 → 设交易对/周期/时间范围（或上传文件）→ 点「加载行情」
2. 配置初始资金、定投金额、均线类型与周期遍历范围、ahr999 阈值
3. 点「运行回测」，查看资金曲线与收益对比

> 部分网络环境对 Binance API 有访问限制，此时改用本地 Excel/CSV 上传。`sample_data.csv` 为示例文件。

## ahr999 指数说明

```
ahr999 = (现价 / 200日定投成本均价) × (现价 / 指数增长估值)
指数增长估值 = 10^(5.84 × log10(币龄天数) − 17.01)   // 币龄自 2009-01-03 创世起算
```

经验区间：< 0.45 抄底区，0.45 ~ 1.2 定投区，> 1.2 偏贵。
